Common Reproducibility Mistakes

Session 6 — Coding & Documentation  ·  Wrong pattern on the left. Click show answer to reveal the correct version.

01 Hard-coded paths

Absolute paths are tied to one machine. They break for every other user immediately.

WRONG
import os
os.chdir("C:/Users/Anna/Project")
data = pd.read_csv("data.csv")
CORRECT 
from pathlib import Path
HERE = Path(__file__).parent
data = pd.read_csv(HERE / "data.csv")
click to show answer
WRONG
setwd("C:/Users/Anna/Project")
data <- read.csv("data.csv")
CORRECT 
library(here)
# here() finds the project root automatically
data <- read.csv(here("data.csv"))
click to show answer

The correct version runs on any machine because the path is relative to the script itself, not to a specific folder on a specific laptop.

02 Hard-coded values scattered through the code

When a value appears in multiple places, changing it means hunting through the whole file — and it's easy to miss one.

WRONG
df    = df[df["year"] == 2019]
title = f"Wages in 2019"
out   = "wages_2019.csv"
CORRECT 
YEAR = 2019  # change here only

df    = df[df["year"] == YEAR]
title = f"Wages in {YEAR}"
out   = f"wages_{YEAR}.csv"
click to show answer
WRONG
df    <- df[df$year == 2019, ]
title <- "Wages in 2019"
out   <- "wages_2019.csv"
CORRECT 
YEAR <- 2019  # change here only

df    <- df[df$year == YEAR, ]
title <- paste("Wages in", YEAR)
out   <- paste0("wages_", YEAR, ".csv")
click to show answer

Keep all configurable values at the top. The rest of the code should never contain a literal year, country name, or threshold — only the variable.

03 Copy-pasted blocks

Repeating the same logic for each country means any fix must be applied in every copy. One bug becomes many bugs.

WRONG
de = df[df["country"]=="Germany"].copy()
de["growth"] = de["gdp"].pct_change() * 100

pl = df[df["country"]=="Poland"].copy()
pl["growth"] = pl["gdp"].pct_change() * 100

fr = df[df["country"]=="France"].copy()
fr["growth"] = fr["gdp"].pct_change() * 100
CORRECT 
COUNTRIES = ["Germany", "Poland", "France"]

df = (df[df["country"].isin(COUNTRIES)]
  .sort_values(["country", "year"])
  .assign(growth=lambda x:
    x.groupby("country")["gdp"]
     .pct_change() * 100)
)
click to show answer
WRONG
de <- df[df$country == "Germany", ]
de$growth <- c(NA, diff(de$gdp)/head(de$gdp,-1)*100)

pl <- df[df$country == "Poland", ]
pl$growth <- c(NA, diff(pl$gdp)/head(pl$gdp,-1)*100)

fr <- df[df$country == "France", ]
fr$growth <- c(NA, diff(fr$gdp)/head(fr$gdp,-1)*100)
CORRECT 
COUNTRIES <- c("Germany", "Poland", "France")

df <- df |>
  group_by(country) |>
  arrange(year) |>
  mutate(growth = (gdp/lag(gdp) - 1) * 100) |>
  ungroup()

Adding a fourth country now costs one line in the configuration — not another copy-pasted block.

04 Repeating logic without functions

When the same calculation appears in multiple places, extract it into a function. One definition, tested once, used everywhere.

WRONG — same formula copied twice
mu = np.average(df["task"], weights=df["share_BE"])
sd = np.sqrt(np.average((df["task"]-mu)**2,
             weights=df["share_BE"]))
df["std_BE"] = (df["task"] - mu) / sd

mu = np.average(df["task"], weights=df["share_PL"])
sd = np.sqrt(np.average((df["task"]-mu)**2,
             weights=df["share_PL"]))
df["std_PL"] = (df["task"] - mu) / sd
CORRECT 
def wtd_std(series, weights):
    mu = np.average(series, weights=weights)
    sd = np.sqrt(np.average(
            (series-mu)**2, weights=weights))
    return (series - mu) / sd

for country in COUNTRIES:
    df[f"std_{country}"] = wtd_std(
        df["task"], df[f"share_{country}"])
click to show answer
WRONG — same formula copied twice
mu <- wtd.mean(df$task, df$share_BE)
sd <- sqrt(wtd.var(df$task, df$share_BE))
df$std_BE <- (df$task - mu) / sd

mu <- wtd.mean(df$task, df$share_PL)
sd <- sqrt(wtd.var(df$task, df$share_PL))
df$std_PL <- (df$task - mu) / sd
CORRECT 
wtd_std <- function(series, weights) {
  mu <- wtd.mean(series, weights)
  sd <- sqrt(wtd.var(series, weights))
  (series - mu) / sd
}

for (country in COUNTRIES) {
  col <- paste0("std_", country)
  df[[col]] <- wtd_std(
    df$task, df[[paste0("share_", country)]])
}

If the formula needs to change, you change it in one place. With copy-paste, a change in one copy and not the others produces silently inconsistent results.

05 Outputs not saved

A plot that only displays is not reproducible. When the session closes, it is gone.

WRONG
plt.plot(df["year"], df["wage"])
plt.title("Real wages")
plt.show()  # disappears on close
CORRECT 
plt.plot(df["year"], df["wage"])
plt.title("Real wages")
plt.savefig(HERE / "figures" / "wages.pdf",
            bbox_inches="tight")
plt.close()
click to show answer
WRONG
p <- ggplot(df, aes(year, wage)) +
  geom_line() + ggtitle("Real wages")
print(p)  # disappears on close
CORRECT 
p <- ggplot(df, aes(year, wage)) +
  geom_line() + ggtitle("Real wages")
ggsave(here("figures", "wages.pdf"),
       p, width=8, height=4)
click to show answer

The same applies to tables — always save to a path built from your project root, not a bare filename.

06 Comments that restate the code

A comment should explain why, not what. A reader can see what the code does — they cannot see the methodological reasoning behind it.

WRONG — restates the code
def wtd_std(series, weights):
    # calculate weighted mean and standard deviation
    mu = np.average(series, weights=weights)
    sd = np.sqrt(np.average(
        (series - mu)**2, weights=weights))
    return (series - mu) / sd
CORRECT — explains the why 
def wtd_std(series, weights):
    """Weighted standardisation (Autor method).
    Weights are employment shares so the mean
    reflects workforce composition, not a simple
    average across occupations.
    """
    mu = np.average(series, weights=weights)
    sd = np.sqrt(np.average(
        (series - mu)**2, weights=weights))
    return (series - mu) / sd
click to show answer
WRONG — restates the code
wtd_std <- function(series, weights) {
  # calculate weighted mean and standard deviation
  mu <- wtd.mean(series, weights)
  sd <- sqrt(wtd.var(series, weights))
  (series - mu) / sd
}
CORRECT — explains the why 
#' Weighted standardisation (Autor method).
#' Weights are employment shares so the mean
#' reflects workforce composition, not a simple
#' average across occupations.
#' @param series  numeric — task scores
#' @param weights numeric — employment shares
wtd_std <- function(series, weights) {
  mu <- wtd.mean(series, weights)
  sd <- sqrt(wtd.var(series, weights))
  (series - mu) / sd
}
click to show answer

"Calculate mean and standard deviation" tells a reader nothing new. "Weights are employment shares so the mean reflects workforce composition" is the kind of thing a future reader — including yourself in six months — actually needs.

07 Silent failures

Code that produces a wrong result without any warning is harder to catch than code that crashes. Check your assumptions explicitly.

WRONG — wrong result, no warning
def gini(wages):
    # silently wrong if wages has < 10 rows
    # or contains negative values
    wages = np.sort(wages)
    n = len(wages); idx = np.arange(1, n+1)
    return (2*(idx*wages).sum())/(n*wages.sum()) - (n+1)/n
CORRECT — fails loudly 
import warnings

def gini(wages):
    if len(wages) != 10:
        warnings.warn(
            f"Expected 10 deciles, got {len(wages)}")
        return None
    if not (wages > 0).all():
        warnings.warn("Non-positive wages — skipping")
        return None
    wages = np.sort(wages)
    n = len(wages); idx = np.arange(1, n+1)
    return (2*(idx*wages).sum())/(n*wages.sum()) - (n+1)/n

gini <- function(wages) {

gini <- function(wages) {
